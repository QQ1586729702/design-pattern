### 代理模式
    代理模式: 通过调用代理函数来调用原函数,在调用过程当中可以加入操作,对原来的函数,参数,返回值等进行增强
[静态代理](./static_proxy.py)

    静态代理可以通过自定义函数来对原函数进行增强
    优势: 灵活性强,每一个方法都可以自定义增强的方式
    劣势: 当一个类中的所有方法都需要增强的时候就很繁琐了,需要很多方法来进行增强处理

[动态代理](./dynamic_proxy.py)
    
    动态代理通过反射函数来让调用原类中的方法
    优势: 可以让需要相同的增强的函数都通过一个方法来调用
    劣势: 灵活性差,有可能有的函数并不需要增强
